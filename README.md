# DC-Walsh Co-authorship Graph

Interactive co-authorship network for the Walsh + Christoffel labs at UNC. Renders the **`wiki/entities/people/`** subset of the [NFB LLM Wiki](https://github.com/) (private sibling repo) as a static site with Obsidian-style graph view, search, and per-person pages.

**Live:** https://greenpolo.github.io/DC-Walsh-Coauthorship-Graph/ (lands on Jessica Walsh — expand the right sidebar for the graph view, or click the graph icon to enter global graph mode).

## How this is built

The site is generated by the [Webpage HTML Export](https://github.com/KosmosisDire/obsidian-webpage-export) Obsidian plugin (v1.9.2) running against `wiki/entities/people/` in the sibling wiki vault. Output is committed to this repo's root and served via GitHub Pages.

### Regenerating after wiki ingests

From within an open Obsidian instance pointed at the wiki vault:

```powershell
# 1. Make sure settings still point at wiki/entities/people/ and the staging dir
obsidian eval code="(async () => { const s = this.app.plugins.plugins['webpage-html-export'].settings; s.exportOptions.filesToExport = ['wiki/entities/people']; s.exportOptions.exportPath = 'C:/WalshLab/webpage-export-output'; s.onlyExportModified = false; s.deleteOldFiles = true; await this.app.plugins.plugins['webpage-html-export'].saveData(s); return 'ok'; })()"

# 2. Run the export
obsidian command id=webpage-html-export:export-html-vault

# 3. Wait for the dir to populate, then swap into the GH Pages repo
cp -r C:/WalshLab/webpage-export-output/. C:/WalshLab/DC-Walsh-Coauthorship-Graph/
cd C:/WalshLab/DC-Walsh-Coauthorship-Graph

# 4. Build the graph-only landing page (overrides index.html)
python make_landing.py

# 5. Commit + push
git add -A && git commit -m "refresh: re-export from wiki" && git push
```

GitHub Pages rebuilds automatically (~30–60s lag).

## What gets exposed

The export bundles each person page's full body — affiliation, role, h-index, ORCID, citation count, co-author wikilinks, and any prose. Source pages and the rest of the wiki are *not* exposed. Frontmatter property view is enabled at the top of each page (the same fields visible in Obsidian's properties UI).

If a person should not have their data on a public site, remove their `.html` from the export output before pushing — or strip the corresponding `wiki/entities/people/<name>.md` of frontmatter before re-running the export.

## Structure

- `index.html` — graph-only landing (Jess's page with sidebar/document/outline hidden via injected CSS, global graph auto-triggered on load via injected JS).
- `*.html` — one per person page (slugified-lowercase basenames). Includes a clean `jessica-j-walsh.html` (the canonical PI page; sidebar + content visible).
- `site-lib/` — fonts, styles, scripts (including the WASM graph renderer), search index, RSS, metadata.
- `make_landing.py` — post-export script that rebuilds `index.html` from `jessica-j-walsh.html` with the graph-only overrides.
- `.gitignore`, `README.md`, `.git/`

~13 MB total, ~280 files.
